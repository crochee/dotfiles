# Repo Wiki 工作流程

本文档定义 AI 生成 wiki 的完整三阶段工作流程。每次生成 wiki 时必须严格遵循。

## 总览

```
Phase 1: Scaffold（脚手架）  →  Phase 2: Analyze（分析）  →  Phase 3: Fill（填充）
```

每个阶段有明确的输入、输出和退出条件。不得跳过任何阶段。

---

## Phase 1: Scaffold — 生成目录结构

**目标**：创建 `.open_docs/` 目录和页面骨架文件。

**输入**：
- 用户查询（用于语言检测）
- 可选：`--lang` 显式语言代码
- 可选：`--input` 外部 JSON 数据

**执行**：
1. 检测或确认目标语言
2. 运行 `scaffold_open_docs.py` 脚本（或等效逻辑）
3. 生成以下结构：
   ```
   .open_docs/
     index.md                     # 首页，含页面列表
     01-overview.md               # 骨架文件
     01-01-quick-start.md
     ...
   ```

**输出**：
- 目录结构完整创建
- `index.md` 含正确页面列表
- 每个页面文件含：H1 标题、导航行、占位提示文本、`<details>` 块

**退出条件**：
- 所有骨架文件已创建
- 每个页面包含占位标记（如"此页为模板脚手架自动生成"）

**注意**：此阶段不涉及任何代码库分析。

---

## Phase 2: Analyze — 代码库分析

**目标**：系统性地分析代码库，产出每个文件和文件夹的摘要。

**输入**：
- 完整的仓库代码
- Phase 1 生成的骨架结构（用于确定需要哪些页面）

**执行**：按照 FETCH → FILTER → SUMMARIZE 三子阶段进行。

### 子阶段 2.1: FETCH — 仓库发现

收集以下五类信息：

1. **元数据源** — 从 README.md、包配置文件提取项目名称、描述、版本、依赖
2. **目录结构** — 映射完整目录树，裁剪噪声目录
3. **入口点** — 按语言识别执行入口（Python/Node.js/Go/Rust）
4. **配置源** — 搜索环境变量引用、配置文件、CLI 参数解析器
5. **数据模型** — 识别模型/实体/模式定义

### 子阶段 2.2: FILTER — 信号提取

1. 排除二进制文件和生成文件
2. 将文件分类为代码/配置/文档
3. 优先分析核心逻辑目录（`src/`、`lib/`、`internal/` 等）

### 子阶段 2.3: SUMMARIZE — 自底向上生成

按顺序处理：

1. **Files first** — 为每个文件生成摘要（用途 + 2-3 段描述 + 代码引用）
2. **Folders deepest-first** — 从叶子文件夹到根文件夹生成摘要，使用子级摘要作为上下文
3. **Root summary last** — 将所有顶级摘要聚合为 Overview 内容

**输出**：
- 每个文件的摘要
- 每个文件夹的摘要
- 文件夹的 Mermaid 依赖图（如适用）

**退出条件**：
- 所有需要分析的源文件已处理
- 根目录摘要已聚合

### 空仓库特殊处理

当仓库无可分析的源代码时：
- 跳过 FETCH → FILTER → SUMMARIZE 流水线
- 直接使用 README 和包元数据生成页面内容
- 每页至少生成 2 段有意义的内容

---

## Phase 3: Fill — 内容填充

**目标**：将分析结果写入骨架文件，替换占位标记为真实内容。

**输入**：
- Phase 1 生成的骨架文件
- Phase 2 产出的分析结果

**执行**：

1. **逐页填充** — 按页面计划顺序处理每个页面
2. **内容写入** — 将分析结果转化为页面内容，遵循页面模板指导
3. **导航更新** — 确保每页导航行正确指向相邻页面
4. **Details 更新** — 用真实文件路径替换占位符

**每页验证**：
- [ ] 无残留占位标记
- [ ] 导航行正确
- [ ] 内容基于实际代码
- [ ] 技术令牌未翻译
- [ ] 语言一致（无混合语言）

**输出**：
- 完整的多页 wiki，每页含真实内容
- `index.md` 含正确页面列表

**退出条件**：
- 所有计划页面已填充
- 质量检查清单全部通过

---

## 页面类型与阶段对应

| 页面类型 | Phase 2 分析深度 | Phase 3 内容来源 |
|----------|-----------------|-----------------|
| Overview | 根目录摘要 | 元数据 + 入口点 + 数据模型 |
| Quick Start | 入口点 + 配置源 | 安装脚本 + Makefile + Dockerfile |
| Project Structure | 完整目录结构 | ls 输出 + 包布局 |
| Configuration | 配置源 | 环境变量引用 + 配置文件 |
| Usage | 入口点 + 测试文件 | README 示例 + examples 目录 |
| Development | 测试配置 + lint 配置 | Makefile + CONTRIBUTING.md |
| CLI Commands | 入口点（CLI） | pyproject.toml scripts + Makefile |
| API Reference | 公共模块 | __all__ + 路由定义 |
| Deployment | 部署配置 | Dockerfile + CI/CD |
| Architecture | 包依赖 | 源码结构 |
| Testing | 测试文件 | 测试配置 + 测试目录 |
| Contributing | 贡献文件 | CONTRIBUTING.md + CODE_OF_CONDUCT |
