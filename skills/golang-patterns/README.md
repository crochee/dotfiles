# Golang Patterns

Idiomatic Go patterns and best practices for building robust applications.

## What This Skill Does

Provides conventions for Go development including error handling, concurrency patterns, interface design, package organization, and performance optimization.

## When It Activates

- Writing new Go code
- Reviewing or refactoring Go code
- Designing Go packages
- Implementing concurrency, error handling, or interface patterns

## Key Topics Covered

- Error wrapping and custom error types
- Concurrency patterns (worker pools, context cancellation, errgroup)
- Interface design (small focused interfaces, define where used)
- Package organization and naming
- Struct design (functional options, embedding)
- Memory and performance optimization
- Anti-patterns to avoid
