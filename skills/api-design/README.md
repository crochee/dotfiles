# API Design

REST API design patterns for building consistent, developer-friendly APIs.

## What This Skill Does

Provides conventions for resource naming, HTTP methods, status codes, pagination, filtering, error responses, versioning, and rate limiting.

## When It Activates

- Designing or reviewing REST API endpoints
- Implementing pagination, filtering, or error handling
- Planning API versioning strategies
- Building public or partner-facing APIs

## Key Topics Covered

- Resource URL structure and naming conventions
- HTTP methods and status code semantics
- Response format (success, collection, error)
- Offset vs cursor pagination
- Filtering, sorting, and sparse fieldsets
- Authentication, authorization, and rate limiting
- Implementation examples in TypeScript, Python, and Go
