# Deep Research Quality Rules

**Load this reference when:** ensuring research quality.

## Source Quality Hierarchy

| Source Type | Priority | Use For |
|--------------|----------|---------|
| Academic papers | Highest | Scientific claims, frameworks |
| Official docs | High | Facts, specifications |
| Reputable news | Medium | Current events, opinions |
| Blog posts | Lower | Perspectives, tutorials |
| Forums | Lowest | Community experiences |

## Fact-Checking Rules

| Rule | Application |
|------|-------------|
| Every claim needs source | No unsourced assertions |
| Cross-reference claims | One source = unverified |
| Recency matters | Prefer < 12 months old |
| Acknowledge gaps | Say "insufficient data" |
| Label estimates | Distinguish fact from inference |

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Single source claims | Cross-reference or flag as unverified |
| Outdated sources | Check dates, prefer recent |
| Vague claims | Be specific: "According to X..." |
| Missing attribution | Inline citations always |
| Assumptions as facts | Label clearly |

## Search Optimization

For best results:
- Use 2-3 keyword variations per sub-question
- Mix general and news-focused queries
- Prioritize official sources for facts
- Use quotes for specific phrases
