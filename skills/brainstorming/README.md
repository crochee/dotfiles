# Brainstorming Skill

Socratic design refinement before writing any code.

## Purpose

This skill ensures requirements are clarified and alternatives are explored BEFORE implementation begins. It prevents the common AI pattern of jumping straight into code without understanding the problem.

## Features

- **Socratic Questioning**: Asks "what" not "how" to understand real problems
- **Alternative Exploration**: Presents 2-3 approaches with explicit trade-offs
- **Chunked Design**: Shows design in digestible sections, waits for validation
- **Design Document**: Saves validated design to `.agents/design.md`

## When to Use

- Starting any new feature
- Requirements are unclear
- Exploring architecture options
- User asks "how should I approach X?"

## Workflow

```
Question → Explore → Present → Validate → Save
```

## Output

- Design document saved to `.agents/design.md`
- Clear problem statement
- Documented trade-offs
- Open questions flagged
