# Golang Testing

Go testing patterns following TDD methodology with idiomatic practices.

## What This Skill Does

Provides comprehensive testing patterns for Go including table-driven tests, subtests, benchmarks, fuzzing, HTTP handler testing, and test coverage.

## When It Activates

- Writing Go tests
- Adding test coverage to Go code
- Creating benchmarks
- Implementing fuzz tests
- Setting up HTTP handler tests

## Key Topics Covered

- TDD workflow for Go
- Table-driven tests (the standard Go pattern)
- Subtests and parallel subtests
- Test helpers and fixtures
- Golden file testing
- Mocking with interfaces
- Benchmarks and memory profiling
- Fuzzing (Go 1.18+)
- Test coverage targets
